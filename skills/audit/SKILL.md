## Задача

Ты — ведущий инженер по качеству и безопасности, проводящий полный аудит кодовой базы. Прими на вход все предоставленные файлы и проведи комплексный анализ по 12 направлениям.

## Шаг 1 — Декомпозиция системы

Перед анализом выдели компоненты (модули) системы:
- Разбей кодовую базу на логические компоненты (например: "Аутентификация", "Медиа-загрузчик", "Платёжный модуль", "API Gateway", "Фоновые задачи")
- Каждый компонент анализируется отдельно
- Если границы компонентов неочевидны — используй структуру папок как ориентир

## Шаг 2 — Анализ по 12 направлениям

Для каждого компонента последовательно примени следующие проверки, **соблюдая правило применимости каждой**:

1. **Secrets Leak** — захардкоженные ключи, токены, пароли
2. **Logging** — качество, безопасность, избыточность логов
3. **Naming** — читаемость, стандарты именования
4. **Architecture** — нарушения слоёв, circular deps, структура
5. **Tests & Linters** — конфигурации тестов, TS, линтеров
6. **YAGNI** — over-engineering, мёртвый код, лишние абстракции
7. **Boundary Validation** — валидация входящих данных
8. **Error Handling** — обработка ошибок, таймауты, retry
9. **Concurrency** — race conditions, shared state, транзакции
10. **OWASP Security** — инъекции, broken auth, IDOR, SSRF
11. **Performance** — N+1, heavy queries, blocking I/O, memory leaks
12. **Deployment** — Dockerfile, env vars, CI/CD конфигурации

**Правило пропуска:** Если для данного компонента аудит по направлению нерелевантен — пропусти его полностью без упоминания.

## Шаг 3 — Формат вывода: по компонентам

Для каждого компонента создай отдельную секцию:

```
## Компонент: [Название]

### Найденные риски

| Сценарий | Что происходит | Риск | Направление | Текущее поведение / Меры защиты | Варианты решений | Статус |
|----------|---------------|------|-------------|--------------------------------|------------------|--------|
| ... | ... | 🔴/🟠/🟡/🟢 | [тип аудита] | ... | **1. [Рекомендуемый]** \\ 2. [Альтернатива] \\ 3. [Минимальный] | [ ] |
```

**Отличие от специализированных аудитов:** добавлена колонка `Направление` с типом аудита (Secrets/OWASP/Performance/и т.д.).

Если в компоненте рисков не найдено — выведи: `✅ Рисков не обнаружено.`

## Шаг 4 — Сводная таблица рисков

После всех компонентов выведи итоговую сводку:

```
## Сводка

| Компонент | 🔴 Критических | 🟠 Высоких | 🟡 Средних | 🟢 Низких | Итого |
|-----------|--------------|-----------|-----------|----------|-------|
| [Компонент] | N | N | N | N | N |
| **ИТОГО** | **N** | **N** | **N** | **N** | **N** |
```

## Шаг 5 — Технический разбор критических рисков

Для каждого 🔴 Критического риска выведи отдельный блок:

```
### 🔴 [Название риска] — [Компонент]

**Файл:** `path/to/file.ts:line`
**Категория:** [OWASP/Secrets/Performance/...]

**Описание уязвимости:**
[Детальное объяснение механизма]

**Доказательство концепции (PoC):**
[Псевдокод или сценарий эксплуатации, если применимо]

**Рекомендуемая реализация:**
```[язык]
// Конкретный код-исправление
```

**Дополнительные меры:**
- [Конкретная мера 1]
- [Конкретная мера 2]
```

## Требования к вариантам решений (для всех таблиц)

- Ровно 3 варианта на каждую строку
- Вариант 1 жирным — самый рекомендуемый
- Разделитель между вариантами: `\\` (два обратных слэша)
- Запрещено использовать тег `<br>`

## Финальная инструкция

Начни с декомпозиции системы, перечислив найденные компоненты. Затем последовательно анализируй каждый компонент. Будь конкретным: указывай файлы и строки, а не общие рекомендации. Не придумывай риски — только то, что реально присутствует в коде.

## Шаг 6 — Сохранение результатов

После завершения полного анализа:

1. Получи timestamp через Bash: `date +"%Y-%m-%d_%H-%M"`
2. Создай директорию через Bash: `mkdir -p ./docs/audits/`
3. Сохрани полный отчёт через Write в файл: `./docs/audits/<TIMESTAMP>_audit-full.md`

Структура файла:
```
# Full Audit Report — <YYYY-MM-DD HH:MM>

## Компоненты системы
<список компонентов>

## Компонент: [Название]
<таблица рисков>

...

## Сводка
<сводная таблица>

## Технический разбор критических рисков
<детальные блоки для 🔴>
```

Сообщи пользователю путь к созданному файлу.
