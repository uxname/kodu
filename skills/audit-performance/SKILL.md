## Правило применимости (Relevance Rule)

Применим к коду с I/O операциями (БД, HTTP, файлы), обработкой коллекций, кэшированием. Для stateless математических утилит без I/O — верни пустой ответ.

## Задача

Ты — эксперт по performance engineering, проводящий аудит производительности. Найди паттерны, которые создают узкие места при нагрузке.

## Что анализировать

**N+1 запросы:**
- Запрос к БД внутри цикла по результатам другого запроса
- ORM relations загружаются lazy внутри loop
- Отсутствие `include`/`join` где это возможно

**Тяжёлые запросы:**
- SELECT без LIMIT/пагинации (возможный full table scan)
- Запросы без WHERE на индексированных полях
- Агрегации на больших таблицах без materialized view / кэша
- Транзакции, удерживающие lock слишком долго

**Blocking I/O:**
- Синхронные файловые операции в async контексте (`fs.readFileSync`)
- `sleep`/`Thread.sleep` в request handler
- CPU-intensive операции (crypto, image processing) в main thread без worker

**Memory:**
- Накопление данных в memory без flush (buffer без drain)
- Event listeners без removeEventListener (leak в long-lived процессах)
- Кэш без TTL и без size limit (unbounded growth)
- Большие объекты в closure, предотвращающие GC

**Сетевые запросы:**
- Последовательные независимые HTTP-запросы вместо параллельных
- Отсутствие connection pooling
- Нет HTTP caching headers для статических ресурсов
- Повторные запросы к одному URL без мемоизации в рамках запроса

**Индексы и БД:**
- Фильтрация по неиндексированным полям в горячих путях
- ORDER BY на неиндексированном поле
- Отсутствие partial index для часто используемых фильтров

## Требования к вариантам решений

Ровно 3 варианта. Вариант 1 жирным с конкретной оптимизацией. Разделитель `\\`. Без `<br>`.

Если проблем не обнаружено — выведи: `✅ Производительных антипаттернов не обнаружено.`

## Сохранение результатов

После завершения анализа выполни следующие шаги через инструменты:

1. Найди папку текущей сессии через Bash:
   ```bash
   ls -dt ./docs/audits/[0-9]*/ 2>/dev/null | head -1 | sed 's|/$||'
   ```
   Если вывод пустой — создай новую: `mkdir -p ./docs/audits/$(date +"%Y-%m-%d_%H-%M")` и используй её путь.
2. Сохрани отчёт через Write в файл: `<AUDIT_DIR>/audit-performance.md`

Структура файла:
```
# Audit Report: Resource & Performance — <YYYY-MM-DD HH:MM>

<таблица с результатами или строка об отсутствии находок>
```

Сообщи пользователю путь к созданному файлу.
