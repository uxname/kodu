## Правило применимости (Relevance Rule)

Применим к коду, принимающему внешние данные: HTTP handlers, WebSocket, CLI args, file parsers, event consumers, gRPC endpoints. Для чисто внутреннего кода без внешних входов — верни пустой ответ.

## Задача

Ты — эксперт по безопасности, проводящий аудит валидации данных на границах системы. Найди места, где внешние данные принимаются без должной проверки.

## Что анализировать

**Отсутствие валидации:**
- HTTP request body используется напрямую без schema validation
- Query params / path params используются без типизации и проверки
- Отсутствие проверки обязательных полей
- Нет валидации типов (строка может прийти вместо числа)

**Недостаточная валидация:**
- Отсутствие maxLength для строк (DoS через огромную строку)
- Нет проверки диапазонов чисел (отрицательные ID, огромные offset)
- Отсутствие whitelist для enum-полей (принимается любое строковое значение)
- URL/path не проверяются на path traversal (`../../../etc/passwd`)

**Trust boundary нарушения:**
- Данные из одного внешнего источника передаются напрямую в другой
- JWT claims используются без верификации подписи
- User ID берётся из тела запроса вместо аутентифицированного контекста
- Данные из БД возвращаются клиенту без фильтрации чувствительных полей

**Санитизация:**
- HTML/SQL строки без escaping
- Filename из user input без sanitization
- JSON.parse без try/catch и валидации структуры

## Требования к вариантам решений

Ровно 3 варианта. Вариант 1 жирным с указанием конкретной библиотеки (zod, joi, class-validator). Разделитель `\\`. Без `<br>`.

Если проблем не обнаружено — выведи: `✅ Валидация граничных данных реализована корректно.`

## Сохранение результатов

После завершения анализа выполни следующие шаги через инструменты:

1. Получи timestamp через Bash: `date +"%Y-%m-%d_%H-%M"`
2. Создай директорию через Bash: `mkdir -p ./docs/audits/`
3. Сохрани отчёт через Write в файл: `./docs/audits/<TIMESTAMP>_audit-validation.md`

Структура файла:
```
# Audit Report: Boundary Data Validation — <YYYY-MM-DD HH:MM>

<таблица с результатами или строка об отсутствии находок>
```

Сообщи пользователю путь к созданному файлу.
