## Технический разбор критических рисков

### 🔴 Path Traversal через config-driven prompt path — Config Module

**Файл:** `src/core/config/prompt.service.ts:29–44`
**Категория:** OWASP A01 (Broken Access Control) / Boundary Validation

**Описание уязвимости:**

`PromptService.readSource` резолвит путь, взятый из `kodu.json → prompts.pack`, через `path.resolve(process.cwd(), source)`. Проверки, что resolved path находится внутри проекта, нет. Если `kodu.json` содержит `"prompts": { "pack": "../../.env" }` или абсолютный путь `/etc/passwd`, метод успешно прочитает файл и вернёт его содержимое. Это содержимое включается в packed context, который пользователь затем отправляет в LLM.

Вектор атаки: злоумышленник публикует open-source проект с crafted `kodu.json`. Жертва клонирует репо, запускает `kodu pack`, содержимое `.env` / SSH-ключей / etc. уходит в LLM-сервис.

**Доказательство концепции (PoC):**

```json
// kodu.json в malicious репо
{
  "prompts": {
    "pack": "../../.ssh/id_rsa"
  }
}
```

```
$ kodu pack
# В .kodu/context.txt появится содержимое ~/.ssh/id_rsa
```

Также работает для `.env`:
```json
{ "prompts": { "pack": "../../../home/user/.env" } }
```

**Рекомендуемая реализация:**

```typescript
// prompt.service.ts
private async readSource(source: string): Promise<string> {
  const resolved = path.isAbsolute(source)
    ? source
    : path.resolve(process.cwd(), source);

  // ✅ Проверка, что путь остаётся внутри проекта
  const projectRoot = process.cwd();
  const relative = path.relative(projectRoot, resolved);
  const isOutside = relative.startsWith('..') || path.isAbsolute(relative);

  if (isOutside) {
    throw new Error(
      `Prompt path "${source}" resolves outside project directory. ` +
      `Only paths within "${projectRoot}" are allowed.`,
    );
  }

  if (await this.exists(resolved)) {
    return this.readAndCache(resolved);
  }

  if (this.looksLikeInline(source)) {
    return source;
  }

  throw new Error(
    `Prompt file not found: ${path.relative(projectRoot, resolved)}`,
  );
}
```

**Дополнительные меры:**

- Ограничить загрузку промптов только из `.kodu/prompts/` (использовать `loadFromPromptsDir` вместо произвольного пути)
- Добавить валидацию в Zod-схему: `prompts.pack` должен быть относительным путём без `..` (`z.string().regex(/^(?!\.\.)/)`)
- Документировать в README, что `prompts.pack` — путь внутри проекта
